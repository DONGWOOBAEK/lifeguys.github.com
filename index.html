Jekyll를 이용한 Github에 블로그 생성

Middle Project lifeguys

1. 개요

지킬(Jekyll)

템플릿 디렉토리에서 다양한 포맷의 원본 텍스트 파일을 읽은 후, Markdown (또는 Textile) 과 Liquid 변환 작업을 수행하여, 모든 웹 서버에 곧바로 게시할 수 있는 완성된 정적 웹사이트를 만들어 낸다. Jekyll 은 GitHub Pages 의 내부 엔진으로 사용되고 있습니다. Jekyll을 사용하여 자신의 프로젝트 페이지나 블로그, 웹사이트를 GitHub에 무료로 호스팅 할 수 있다.

2. Jekyll 설치

Step 1: Rvm 설치 
sudo apt-get install curl
curl -sSL https://get.rvm.io | bash -s stable
Step 2: RVM환경 설정 
source /home/계정명/.rvm/scrips/rvm
Step 3: 가장 최근 버전의 ruby설치 
rvm requirements
rvm install 2.1.2
Step 4: 기본 버전 설정 
rvm use 2.1.2 --defalut
Step 5: nodejs 설치 
sudo apt-get install nodejs
Step 6: Jekyll 설치 
gem install jekyll
jekyll -v
Step 7: 블로그 만든뒤 서버에 등록 
jekyll new 블로그이름
cd /블로그이름
jekyll server
사이트링크 : http://sharadchhetri.com/2014/06/30/install-jekyll-on-ubuntu-14-04-lts/ 
깜짝 퀴즈!

3. Github 가입 및 Repository 생성

Step 1: 설치

맥 OS X에는 루비가 이미 설치되어 있으므로 터미널에서 ruby -v를 입력하여 루비 버전을 체크한 후 다음 명령으로 지킬을 설치한다. 

sudo gem install jekyll
[sudo]가 필요한 경우엔 []없이 sudo gem install jekyll을 입력한다. 만약, RubyGems 버전과 관련한 에러가 나면 sudo gem update --system으로 업데이트한 후 설치하면 대부분 해결할 수 있다.
이제 로컬 디렉토리에 지킬을 설치하고 온라인 깃허브에 자신의 블로그를 올릴 차례이다. 이 블로그 주소와 같이 nolboo.github.io 계정주소로 생성하는 경우와 nolboo.github.io/blog처럼 프로젝트 페이지 주소로 생성하는 경우가 있다. 전자의 경우가 훨씬 사용하기 쉽다.

Step 2: 계정 주소 페이지

jekyll new 깃허브사용자명.github.com // 깃허브사용자명.github.com 
 디렉토리를 만들어 기본 디렉토리와 화일을 생성한다.
깃허브사용자명.github.com으로 만들면 별도의 비용없이도 자신의 블로그 주소를 가질 수 있다. 자신의 경우에 맞게 깃허브사용명을 입력한다. 
cd 깃허브사용자명.github.com
jekyll serve --watch
이제 웹브라우저 주소창에 localhost:4000을 입력하면 내 컴퓨터(로컬)에서 디폴트 페이지로 생성된 지킬 사이트를 볼 수 있다. 깃허브에서 깃허브사용자명.github.com이란 이름으로 새로운 온라인 저장소를 만든다. 
git init
git remote add origin 저장소URL
git add .
git commit -m "Initialize blog"
git push origin master
Step 3: 프로젝트 주소 페이지 
jekyll new blog // blog 디렉토리를 만들어 기본 디렉토리와 화일을 생성한다.
cd blog
jekyll serve --watch

git init
git checkout --orphan gh-pages
깃허브에 blog란 이름으로 새 저장소를 만들고(로컬 디렉토리와 같은 이름으로 만드는 것이 나중에 기억하거나 유지보수하기가 좋다.) 저장소url을 복사한다 
git remote add origin 저장소url
git add .
git commit -m "Initialize blog"
git push origin gh-pages  // 반드시 gh-pages 브랜치로 푸시해야 깃허브가 블로그 페이지를 만들어 준다.

4. 계정주소 페이지 만들기

Step 1: 프로젝트 폴더 생성 
jekyll new 폴더이름
cd 폴더이름
jekyll server --watch
이후 로컬에서 자신이 만든 블로그를 확인 할 수 있습니다. 
localhost:4000
Step 2: 자신의 깃허브를 통해 웹상으로 이동

git을 설치 한 뒤, git을 초기화 한다. 

sudo apt-get install git
git init
현재 생성한 폴더에 내용을 원격 저장소에 저장하기 위해서 저장소의 주소를 git에 추가한다. 
git remote add origin 저장소 주소
저장소 주소는 생성된 저장소로 들어가면 확인 할 수 있다.

다음으로는 로컬에서 만든 블로그를 github Repository에 추가하기 위한 준비작업을 한다. 

git add .
git commit -m "부가 설명을 위한tag"
여기서 -m은 커밋할 때 어떤 내용인지 메시지를 입력하는 옵션이다.

로컬에서 만든 저장소에 추가

git push origin master
